открыл Console/GTEST.cbp 
для удобства предлагаю для каждой тестируемой функции создать свой .h файл и положить его в Console/gtest 
  и называть его соответственно "принадлежность_функция.h" (пример core_polygon_size.h)
тесты пишутся так:
  TEST(name_class_test,name_test){...}
name_class_test - название группы тестов будем называть их так (как файл) "принадлежность_функция" (пример core_polygon_size)
name_test - имя  теста в этой группе (в одной группе ровно один тест с таким названием должен быть) 
  название тоже лучше "говорящее" (пример TEST(core_polygon_size,1) - проверяет, что Polygon.size() у объекта с одной точкой ==1)
какие функции assert есть в google test расписывать не буду, лучше самостоятельно загуглить
